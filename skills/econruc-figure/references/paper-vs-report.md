# Paper vs Report Figures

## Paper figures

Purpose: convince reviewers that evidence is credible.

Style:

- Compact.
- Minimal title or no title if caption carries title.
- Black/gray plus one accent color.
- Confidence intervals visible and readable.
- Axis labels include units.
- Figure note includes design details.
- Export PDF/SVG with editable text.

Avoid:

- Large decorative title.
- Icons and pictograms.
- Unexplained annotations.
- Multiple palettes in one figure.

Multi-indicator rule:

- If the user asks for related indicators in one sentence, keep them in one figure when possible.
- Use coordinated panels when units differ, such as GW and TWh.
- Use one shared time axis for time-series panels.
- Use one legend and repeated colors for the same concepts across panels.
- Avoid dual axes unless necessary; aligned panels are usually clearer and more credible.

## Report figures

Purpose: help decision makers understand a takeaway quickly.

Style:

- Strong title and subtitle.
- More explicit annotation.
- Data source and definition in footnote.
- Color can guide attention.
- Label major policy dates, regions, or groups.

Avoid:

- Regression-code jargon in title.
- Dense confidence intervals when audience needs direction only.
- Hiding uncertainty if the conclusion depends on it.

Multi-indicator rule:

- When a request combines several indicators, make one integrated report figure with a common title and coherent visual hierarchy.
- Use panel titles such as `A. Capacity` and `B. Generation` or Chinese equivalents.
- Keep annotations sparse and aligned across panels.
- Make the color mapping stable across panels, for example blue for wind and gold for solar in both capacity and generation.

## Slide figures

Purpose: one message per slide.

Style:

- Larger text.
- Direct labels.
- Fewer categories.
- Use callout only for the key estimate or period.

For multi-indicator slide figures, prefer two large coordinated panels or one simplified composite. Drop secondary gridlines, long notes, and small categories.

## Diagnostic figures

Purpose: reveal design threats.

Style:

- Honest over beautiful.
- Label pre-period, threshold, missingness, leverage, outliers.
- Include raw patterns when possible.
