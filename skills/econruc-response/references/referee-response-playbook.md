# Referee Response Playbook

## Classification

Classify each comment as:

- identification
- robustness
- mechanism
- data/sample
- measurement
- literature
- exposition
- contribution
- policy interpretation
- external validity
- table/figure presentation

## Response structure

Use:

1. Thank and restate the concern.
2. Explain what was changed or why no change was made.
3. Report the result if new analysis was run.
4. Point to manuscript location.
5. State remaining boundary if needed.

Template:

`We thank the referee for raising this point. The concern is that [concern]. In response, we [action]. The new results show [result]. We have revised [section/table/page] to make this clearer. We also note that [boundary].`

## Tone rules

- Say `we agree` only when you actually accept the point.
- Say `we clarify` for exposition changes.
- Say `we add` for new analysis.
- Say `we now discuss` for limitations.
- Avoid `clearly`, `obviously`, `as already shown`, `the referee misunderstands`.

## No-change response

Use only when justified:

`We considered this suggestion carefully. We do not implement it because [reason: data unavailable, design would change estimand, request would condition on post-treatment variable]. We have added text explaining this limitation and why the baseline specification remains aligned with the paper's estimand.`

## Chinese author note

When helping a Chinese author team, include a private-facing Chinese strategy note:

- 这个意见真正担心的识别问题是什么
- 需要补哪张表或哪段文字
- 哪些说法必须降调
- 哪些承诺不能写进回复
